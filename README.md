## &#9670; Setup / Installation

This profile README is built from **original SVG assets** (no external SVG downloads) plus a small set of well-known external stat widgets (GitHub Stats, Streak Stats, Top Langs, Activity Graph, Trophy, Contribution Snake) that are conventionally served as live images and re-render automatically — these cannot be "owned" as static files since they reflect your live GitHub data.

### 1. Repository name

GitHub only renders a profile README on a **special repository that matches your username**. Create (or use) a repo named exactly:

```
maneshrava/maneshrava
```

If it doesn't exist yet: on GitHub, click **New repository** → name it `maneshrava` → check **"Add a README file"** → set it **Public**.

### 2. Folder structure

Push this exact structure to the root of that repository:

```
maneshrava/
├── README.md
├── assets/
│   ├── hero-banner.svg
│   ├── divider.svg
│   ├── terminal.svg
│   ├── about-card.svg
│   ├── skills-grid.svg
│   ├── project-cybersentinel.svg
│   ├── project-closetiq.svg
│   ├── project-future.svg
│   ├── roadmap.svg
│   ├── timeline.svg
│   ├── achievements-card.svg
│   ├── contact-card.svg
│   └── footer-wave.svg
└── .github/
    └── workflows/
        └── snake.yml
```

### 3. Push it

```bash
git clone https://github.com/maneshrava/maneshrava.git
cd maneshrava
# copy README.md, assets/, and .github/ into this folder
git add .
git commit -m "Add ocean-blue animated profile README"
git push origin main
```

### 4. Enable the contribution snake

The workflow in `.github/workflows/snake.yml` uses [`Platane/snk`](https://github.com/Platane/snk) to turn your contribution graph into an animated snake, then publishes it to an `output` branch.

- No extra secrets are needed — it uses the automatically-provided `GITHUB_TOKEN`.
- Go to **Settings → Actions → General → Workflow permissions** and set it to **"Read and write permissions"** so the action can push to the `output` branch.
- Run it once manually: **Actions tab → Generate Contribution Snake → Run workflow**.
- After the first successful run, an `output` branch appears containing `github-contribution-snake.svg`, which the README already links to.

### 5. Customize

| What | Where |
|---|---|
| Colors | Edit the gradient `<stop>` hex values inside any file in `assets/` — the full palette (`#020617`, `#0F172A`, `#1E3A8A`, `#2563EB`, `#3B82F6`, `#38BDF8`, `#7DD3FC`, `#E0F2FE`) is reused consistently |
| Project cards | Duplicate `project-future.svg` as a template for new projects |
| Skills | Edit the `<text>` lists inside `skills-grid.svg` |
| Stats theme | Query params on the `github-readme-stats` / streak / activity-graph URLs already match the palette — swap `bg_color`, `title_color`, etc. if you want different accents |

### Why some sections use live external images

GitHub sanitizes README HTML/CSS heavily — `<style>` blocks with animations and `backdrop-filter` glassmorphism do **not** render in raw Markdown. The workaround used throughout this README (and in every well-known animated GitHub profile) is to reference **standalone `.svg` files as `<img>` sources**: GitHub serves those as static image resources, which _do_ support internal SMIL/CSS animation, gradients, and blur filters — that's how every animated effect here (waves, particles, glow, blinking cursor, progress bars, pulsing timeline dots) actually works. GitHub Stats, Streak, Top Langs, Activity Graph, Trophy, and the Contribution Snake are the one exception: they must be live-generated services (or a self-hosted Action, as with the snake) because they reflect real, changing GitHub data — no static SVG can do that.
